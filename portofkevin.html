Proyek

Inventory Management System 2025 – 2026

Javinci Inventory Management
— PT. Javinci Berkat Kreatif Sentosa

Sistem ERP inventori end-to-end: dari Purchase Order hingga produksi dengan Bill of Material dan SPK.

Gambaran Sistem

Saya merancang dan mengimplementasikan delapan modul terintegrasi pada platform web Javinci Inventory Management — sebuah sistem manajemen inventori yang digunakan secara aktif di gudang dan divisi produksi perusahaan kosmetik ini.

Modul yang Dikembangkan

1. Laporan Stok Min/Max

Filter per gudang dan tipe laporan (MIN, MAX, Semua). Export ke Excel dan Print Report.

2. Laporan Summary Mutasi Barang

Rekap pengiriman antar gudang dengan filter status: Gantung, Selesai, Batal, Semua Status.

3. QC Bahan Baku (CRUD)

Manajemen Quality Control lengkap: Create, Detail, Edit, Approve, dan Batalkan QC dengan konfirmasi modal.

4. Upload PDF Resi

Sistem baca nomor resi otomatis dari PDF untuk mencegah double print. Deteksi duplikat, tracking user & timestamp.

5. Purchase Order (CRUD)

Pembuatan PO dengan kalkulasi otomatis Sub Total, Tax, dan Grand Total. Status: Open → Approved → Completed.

6. Laporan PO Detail

Filter tiga dimensi: Status PO × Supplier × Gudang. Menampilkan progress penerimaan per item dengan persentase.

7. Bill of Material (CRUD)

Manajemen BOM untuk tipe Ingredient dan Filling & Packaging. Digunakan sebagai acuan otomatis pada SPK.

8. Surat Perintah Kerja (SPK)

Alur produksi lengkap: Draft → Approved → In Progress → Completed. BOM terisi otomatis saat memilih barang setengah jadi.

Alur Purchase Order → Penerimaan Barang

01

Buat Purchase Order

Input nomor PO, supplier, gudang tujuan, dan item barang via Ajax autocomplete. Grand Total dihitung otomatis termasuk pajak.

02

Approve PO

Manajemen mengonfirmasi PO. Status berubah menjadi Completed dan tidak dapat diubah kembali.

03

Penerimaan Barang Otomatis

Saat membuat penerimaan barang, user cukup memilih nomor PO — form Supplier, Gudang, dan detail item terisi otomatis.

04

Laporan Progress

Laporan PO Detail menampilkan persentase progress penerimaan per item dan per PO secara real-time.

Alur Produksi SPK

01

Draft SPK

User membuat SPK, memilih barang setengah jadi dan barang jadi. Daftar bahan baku dan bahan pengemas terisi otomatis dari BOM.

02

Approve SPK

SPK dikunci dan tidak dapat diedit. Sistem menampilkan tabel Material Kurang (shortage) jika stok tidak mencukupi.

03

Start Production

Status berubah ke In Progress. Material dikonsumsi dari inventory secara otomatis oleh sistem.

04

Complete Production

Produksi selesai. Barang jadi masuk ke inventory. SPK tidak dapat diubah lagi.

Machine Learning · Skripsi 2026

Rice Disease Classifier
— Deteksi Penyakit Daun Padi

Aplikasi web berbasis Django untuk klasifikasi penyakit daun padi menggunakan Random Forest + GLCM + LBP + Color features, akurasi 92.4%.

Gambaran Sistem

Sistem klasifikasi penyakit daun padi berbasis web yang memungkinkan petani dan penyuluh pertanian mengidentifikasi jenis penyakit hanya dengan mengunggah foto daun padi. Model dilatih dengan algoritma Random Forest menggunakan 92 fitur gabungan (GLCM + Color + LBP) dan teknik oversampling BorderlineSMOTE.

Pipeline Ekstraksi Fitur (92 Fitur Total)

GLCM

24 Fitur Tekstur (Gray-Level Co-occurrence Matrix)

Contrast, Dissimilarity, Homogeneity, Energy, Correlation, ASM — masing-masing pada 4 sudut (0°, 45°, 90°, 135°).

Color

39 Fitur Warna (HSV + LAB + Histogram BGR)

HSV 9 fitur, LAB 6 fitur, Histogram 8-bin per channel BGR (24 fitur) — efektif membedakan warna kuning, coklat, hijau pada setiap penyakit.

LBP

29 Fitur Pola Lokal (Local Binary Pattern)

Histogram LBP 26 bin + Mean, Std, Entropy — menangkap pola tekstur lokal pada bercak penyakit.

Fitur Unggulan Aplikasi

  • Validasi gambar berlapis — sistem otomatis menolak gambar yang bukan daun padi (poster, hewan, dokumen) menggunakan analisis warna HSV, MSER, dan bentuk kontur.
  • Upload foto atau ambil gambar langsung dari kamera perangkat di halaman Foto & Deteksi.
  • Hasil deteksi menampilkan confidence score per kelas penyakit dalam persentase.
  • Riwayat diagnosis dengan filter per penyakit, rentang tanggal, confidence minimal, dan nama file.
  • Feedback pengguna — user dapat menilai kebenaran setiap prediksi.
  • Admin Dashboard — statistik global, kelola pengguna, confusion matrix, dan ROC Curve interaktif.
  • Halaman Info Penyakit — deskripsi, gejala, penyebab, dan penanganan per jenis penyakit.
  • Black Box Testing: 31 skenario user + 16 skenario admin — semua berhasil.

Metodologi Pengembangan

01

Requirement

Analisis kebutuhan fungsional dan non-fungsional. Input format JPG/JPEG/PNG, output 4 kelas penyakit beserta confidence score.

02

Design

Pemodelan UML (Use Case, Activity, Sequence, Class Diagram, ERD) dan wireframe antarmuka pengguna.

03

Coding / Implementation

Django 5.2 dengan arsitektur Class-Based Views. Model dilatih dengan GridSearchCV + Stratified K-Fold 5-fold.

04

Testing

Black Box Testing 47 skenario total. Seluruh fitur berjalan sesuai spesifikasi dengan status Berhasil.
